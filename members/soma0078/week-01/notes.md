# Week 1 학습 노트

> React Native 첫 실행 & 기본 개념

## 📚 학습한 내용

### 1. React Native 아키텍처의 이해

- **Bridge vs JSI**: 과거의 비동기 JSON 브릿지 방식에서 벗어나, JavaScript가 C++ 객체를 직접 참조하는 **JSI(JavaScript Interface)** 방식으로 진화함. 이로 인해 불필요한 직렬화 비용이 사라지고 네이티브 통합 성능이 비약적으로 향상됨.
- **렌더링 파이프라인**: Fabric 렌더러와 Yoga 레이아웃 엔진을 통해 JSX 구조가 네이티브 뷰(`UIView`, `android.view`)로 동적으로 마운트되는 과정을 이해함

### 2. 개발 환경 전략: Expo vs CLI

- **Expo**: 복잡한 네이티브 설정 없이 빠른 프로토타이핑이 가능하지만, 네이티브 모듈 커스텀에 제약이 있음. 입문 및 표준 앱 개발에 적합함
- **CLI**: OS별 개발 도구(Xcode, Android Studio)를 직접 다루며 높은 자유도를 제공함. 특정 네이티브 라이브러리 연동이나 앱 최적화가 필수적인 환경에서 사용함

### 3. 네이티브 UI 컴포넌트 핵심 요약

- **웹과의 차이**: 추상적인 `div` 대신 레이아웃 목적의 `View`와 텍스트 전용 `Text`를 엄격히 구분함
- **성능 중심의 리스트**: 데이터 양에 따라 `ScrollView`와 레이지 로딩 방식의 `FlatList`를 선택하는 기준을 정립함
- **사용자 경험**: `Button`의 플랫폼별 디자인 불일치를 해결하기 위해 `Pressable`을 활용한 커스텀 상호작용 구현 방식을 익힘

## 💡 인사이트

- 웹보다 플랫폼별 특성(Notch, 키보드 등) 고려가 핵심임
  - 스마트폰 화면 상단에서 카메라, 스피커, 센서 등을 배치하기 위해 실제 디스플레이가 없는 Notch 영역이 존재함

## ❓ 궁금한 점

- `StyleSheet` 내부에서 동작하는 유닛(Unit)이 실제 픽셀(Pixel)과 어떻게 매칭되는지, 디바이스 밀도에 따른 계산 방식의 세부 원리
  - **DP(Density-independent Pixel)**: 모든 수치는 기기 밀도에 상관없이 물리적 크기가 일정하게 보이도록 설계된 가상 단위(DP)를 사용함
  - **계산 방식**: `물리적 픽셀(px) = 논리적 단위(dp) × 기기 밀도(Density)`. 시스템이 `PixelRatio`를 기반으로 자동 계산하여 네이티브 뷰로 전달함
