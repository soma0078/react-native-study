## React Native 주요 컴포넌트

React에서는 HTML 요소를 사용하지만, React Native는 각 플랫폼(iOS, Android)의 네이티브 컴포넌트를 호출하여 렌더링함

```jsx
// Web (React)         →  Native (React Native)
// <div>               →  <View>
// <p>, <h1>, <span>   →  <Text>
// <button>            →  <Button>, <Pressable>
// <img>               →  <Image>
// <input>             →  <TextInput>
// <ul>, <li>          →  <FlatList>, <SectionList>
// <div overflow>      →  <ScrollView>
```

### 1. 기본 레이아웃 컴포넌트

- **`<View>`**: 레이아웃의 기본 단위 (기본 방향이 세로라는 점이 웹과의 가장 큰 차이)
- **`<Text>`**: 웹에서는 div 안에 글자를 써도 되지만, RN에서 텍스트는 반드시 이 태그 안에 있어야 함

### 2. 리스트 및 스크롤 컴포넌트

- **`<ScrollView>`**: 모든 자식 요소를 한 번에 렌더링함. 데이터가 적고 고정적일 때 한 번에 다 보여줘도 성능에 무리가 없을 때 (예: 설정 페이지, 약관 보기)
- **`<FlatList>`**: 화면에 보이는 부분만 메모리에 올리고 나머지는 레이지 로딩(Lazy Loading)함. 데이터가 많거나 늘어날 가능성이 있을 때 성능상 필수적임
- **`<SectionList>`**: 데이터를 섹션별(기능별, 날짜별 등)로 나누어 헤더와 함께 렌더링할 때 사용. 연락처처럼 'ㄱ, ㄴ, ㄷ' 혹은 '날짜별'로 그룹화된 데이터를 표시할 때 유용함

### 3. 사용자 입력 및 상호작용

- **`<Pressable>`**: 스타일링의 자유도가 매우 높음 (`pressed` 상태에 따른 동적 스타일링 가능)
- **`<Button>`**: 빠르고 간단하게 만들 때 쓰지만, iOS/Android 디자인이 달라 통제하기 어려움

### 4. 유용한 레이아웃 컴포넌트

- **`<RefreshControl>`**: `ScrollView`나 `FlatList` 내부에 장착하여 '당겨서 새로고침' 기능을 구현
- **`<KeyboardAvoidingView>`**: 키보드가 올라왔을 때 입력창이 가려지지 않도록 뷰를 자동으로 조정함
