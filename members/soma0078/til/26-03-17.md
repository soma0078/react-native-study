## FlatList vs ScrollView 비교

React Native에서 리스트를 구현할 때 사용하는 두 컴포넌트의 핵심 차이점 정리

### 주요 차이점 요약

| 구분            | FlatList                                                | ScrollView                              |
| :-------------- | :------------------------------------------------------ | :-------------------------------------- |
| **렌더링 방식** | **가상화(Virtualization)**: 화면에 보이는 부분만 렌더링 | 모든 자식 요소를 한 번에 렌더링         |
| **성능/메모리** | 낮음 (네이티브 뷰 객체 재사용)                          | 높음 (아이템 수에 비례하여 증가)        |
| **적합한 사례** | 대량의 데이터, 동적 리스트 (피드, 검색 결과)            | 소량의 데이터, 정적 레이아웃 (설정, 폼) |
| **주요 기능**   | Header/Footer, 무한 스크롤, Pull-to-refresh             | 기본 스크롤 기능                        |

### 코드 비교

<carousel>
```tsx
// FlatList
<FlatList
  data={data}
  renderItem={({ item }) => <RecipeItem item={item} />}
  keyExtractor={(item) => item.id.toString()}
  ListHeaderComponent={<Text>Dinners</Text>}
  showsVerticalScrollIndicator={false}
/>
```
<!-- slide -->
```tsx
// ScrollView
<ScrollView showsVerticalScrollIndicator={false}>
  {data.map((item) => (
    <RecipeItem key={item.id} item={item} />
  ))}
</ScrollView>
```

### FlatList의 메모리 효율성 원리

- **JS 영역**: 전체 데이터 배열은 두 방식 모두 메모리에 동일하게 존재함
- **Native 영역**: `FlatList`는 실제 화면에 보이는 **'네이티브 뷰 객체'**의 개수를 최소한으로 유지하고 이를 재사용함. 반면 `ScrollView`는 화면 밖의 데이터까지 모두 뷰 객체로 만들기 때문에 데이터가 많을수록 메모리 부하가 커짐

### 가상화(Virtualization) vs 무한 로딩(Infinite Loading)

- **가상화**: **메모리 최적화**를 위해 **화면에 보이는 만큼만 렌더링**하는 기술 (FlatList 기본 기능)
- **무한 로딩**: **네트워크 최적화**를 위해 **리스트 끝에 도달할 때 데이터를 추가 fetch**하는 패턴 (`onEndReached`로 구현)
