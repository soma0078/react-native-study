# 2026-03-25 (수) TIL

> Expo(React Native) 프로젝트에서 안드로이드와 iOS 네이티브 앱 빌드 및 에뮬레이터/시뮬레이터 실행 환경(Android Studio, Xcode)을 구축하고 실행하는 방법 정리

## iOS 네이티브 앱 개발 환경

### 1. 사전 준비

Xcode에서 프로젝트를 실행하기 전에 필요한 환경 세팅

1. **Xcode 설치**: App Store에서 Xcode 설치
2. **Command Line Tools 설치**: 시뮬레이터 실행을 위해 필수
   ```bash
   xcode-select --install
   ```
3. **프로젝트 의존성 설치**
   ```bash
   npm install
   ```
4. **iOS 네이티브 폴더 생성 (선택)**: `ios/` 폴더가 없는 경우 아래 명령어로 생성
   (💡 단순히 `run:ios`로 에뮬레이터를 실행할 때는 자동 생성되므로 필수 단계는 아님. 직접 Xcode로 열기 위해 폴더가 필요한 경우에만 수행)

```bash
npx expo prebuild --platform ios

# ios, android 모두 생성
npx expo prebuild
```

---

### 2. 실행 순서

사전 준비가 완료된 후 실제로 앱을 빌드하고 실행하는데 두 가지 방법이 있음

#### 방법 A: Xcode GUI를 활용한 실행 (수동 네이티브 빌드)

Xcode에서 직접 네이티브 코드를 다루거나 디버깅할 때 사용하는 방법

**1단계: CocoaPods 설치 (네이티브 의존성 연결)**

```bash
cd ios
pod install
cd ..
```

**2단계: Xcode로 프로젝트 열기**

```bash
open ios/<project-name>.xcworkspace
```

**3단계: 빌드 및 실행 (Cmd + R)**

1. Xcode 상단의 Scheme 영역에서 `<project-name>`을 선택
2. Scheme 영역 우측 디바이스 목록에서 실행하고자 하는 시뮬레이터 선택
3. `Cmd + R`을 누르거나 `▶` (Play) 버튼을 클릭하여 앱을 빌드하고 실행

**4단계: Metro 번들러 실행 (별도 터미널)**
JS 번들을 서빙하기 위해서는 별도의 터미널 탭에 Metro 번들러를 띄워두어야 접속할 수 있음

```bash
npx expo start
```

---

#### 방법 B: 터미널 명령어를 활용한 단일 실행

터미널 명령어 하나로 빌드부터 에뮬레이터 실행, Metro 번들러 실행까지 한 번에 처리하는 편의 기능

```bash
# iOS 시뮬레이터로 네이티브 앱 빌드 및 실행
npx expo run:ios

# 또는
npm start
i
```

> 💡 위 명령어를 사용하면 `CocoaPods` 자동 동기화, 네이티브 빌드 수행, 시뮬레이터 동작, Metro 번들러 실행이 한 번에 자동으로 이루어지므로, 이 경우에는 **방법 A의 단계별 명령어를 칠 필요가 없음**

### 3. 주의사항 및 트러블슈팅

| 발생하는 상황                       | 해결 방법                                                |
| :---------------------------------- | :------------------------------------------------------- |
| **`ios/` 폴더가 없음**              | `npx expo prebuild` 명령어 실행                          |
| **빌드 에러 (pod 관련 이슈)**       | `cd ios` 이동 후 `pod install` 재실행                    |
| **JS 코드 변경사항이 앱에 미반영**  | Metro 번들러(`npx expo start`) 터미널이 켜져 있는지 확인 |
| **새로운 패키지 추가 후 에러 발생** | `pod install` 다시 실행 후 Xcode에서 클린 빌드(재빌드)   |

> [iOS 시뮬레이터 설치 및 사용 가이드](https://docs.expo.dev/workflow/ios-simulator/)

## Android 네이티브 앱 개발 환경

### 1. 사전 준비

Android Studio에서 프로젝트를 실행하기 전에 필요한 기본 환경 세팅

1. **Android Studio 설치**: 공식 홈페이지에서 다운로드 및 설치
2. **Android SDK 및 AVD 설정**: 최초 실행 시 SDK Manager를 통해 최신 SDK를 설치하고, Device Manager에서 안드로이드 가상 기기(AVD) 생성
3. **환경 변수 설정**: `ANDROID_HOME` 환경 변수를 설정하여 터미널에서 SDK 도구들에 접근할 수 있도록 구성
   - `adb version` 입력했을 때 버전 정보가 나오면 성공
4. **Android 네이티브 폴더 생성 (선택)**: 루트의 `android/` 폴더가 없다면 아래 명령어로 생성
   (💡 단순히 `run:android`로 에뮬레이터를 실행할 때는 자동 생성되므로 필수 단계는 아님. Android Studio로 직접 열기 위해 폴더가 필요한 경우에만 수행)
   ```bash
   npx expo prebuild --platform android
   ```

---

### 2. 실행 순서

안드로이드 기반 프로젝트를 실행하는데 두 가지 방법이 있음

#### 방법 A: Android Studio GUI를 활용한 실행 (수동 네이티브 빌드)

직접 네이티브 코드를 수정하거나 디버깅해야 할 때 사용하는 방법

**1단계: Android Studio로 프로젝트 열기**

Android Studio를 실행하고 React Native 프로젝트 안의 `android/` 폴더 열기

**2단계: 기기 연결 및 빌드 (Shift + F10)**

1. 에뮬레이터를 사용하려면 Device Manager에서 AVD를 재생 버튼으로 실행
2. 상단의 Run 버튼(▶)을 클릭하거나 `Shift + F10`을 눌러 앱을 빌드하고 실행 -> 좀 느림...

**3단계: Metro 번들러 실행 (별도 터미널)**

JS 코드 변경 사항을 앱에 반영하기 위해, 프로젝트 루트 경로의 별도 터미널에서 서빙용 번들러를 켜야 함

```bash
npx expo start
```

---

#### 방법 B: 터미널 명령어를 활용한 단일 실행

터미널 명령어 하나로 안드로이드 빌드와 에뮬레이터 구동을 한 번에 끝내는 쉽고 빠른 방식

```bash
# Android 에뮬레이터로 네이티브 앱 빌드 및 실행
npx expo run:android

# 또는
npm start
a
```

> 💡명령어 실행 전 Android 에뮬레이터가 켜져 있거나 기기가 연결되어 있는지 확인. 빌드 완료 후 자동으로 Metro 번들러 기능이 터미널에 켜지므로, **방법 A 방식대로 따로따로 열 필요 없음**

### ⚠️ 에뮬레이터나 실기기 인식 불가 시

터미널에서 `Error: adb: device offline` 에러 발생 시 `adb devices` 명령어로 연결 상태 확인

1. ADB 서버 초기화

```bash
# 서버 종료
adb kill-server

# 서버 시작
adb start-server

# 연결 상태 확인
adb devices
```

2. Android Studio VDM 실행
   - 경로: Android Studio > Virtual Device Manager > Device Manager
   - 실행하고자 하는 기기 ▶ 클릭

#### ⚠️ 안드로이드 빌드 에러 (Cause: error=2, No such file or directory)

```bash
# Error message
Cause: error=2, No such file or directory
A problem occurred evaluating settings 'android'.
A problem occurred starting process 'command 'node''
```

1. node 실행 경로 지정

```bash
# android/gradle.properties 파일

# Node 실행 경로 강제 지정 (`which node` 결과 경로로 수정)
org.gradle.project.reactNativeNodeExecutable=/usr/local/bin/node
```

2. JDK 17 설치 및 환경 변수 설정

```bash
# 설치
brew install openjdk@17

# 파일 열기
nano ~/.zshrc

# 내용 추가
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export PATH=$JAVA_HOME/bin:$PATH

# 적용
source ~/.zshrc
```

3. Android Studio에서 Gradle JDK를 JDK 17로 지정
   - 경로: Android Studio -> Settings > Build > Build Tools > Gradle > Gradle JDK
   - Gradle JDK: `homebrew-17` 적용
