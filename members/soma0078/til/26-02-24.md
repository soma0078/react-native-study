## 2026-02-24 (화) TLI

### 오늘 배운 것

- 네이티브 앱
  각 운영체제(OS)에 최적화된 전용 언어로 개발된 앱
  - 안드로이드: Java / Kotlin
  - iOS: Objective-C / Swift
  - → 성능과 OS 기능 활용에 강점이 있음
- React Native
  - JavaScript와 React 문법을 사용해 iOS와 안드로이드 앱을 동시에 개발할 수 있는 크로스플랫폼 프레임워크
  - → 하나의 코드베이스로 여러 플랫폼을 개발 가능
- 브릿지(Bridge) 방식의 동작 원리 (Legacy Architecture)
  - JavaScript로 작성한 컴포넌트 로직이 브릿지를 통해 각 OS의 네이티브 모듈과 통신함 (JSON 직렬화 및 비동기 통신을 거침)
  - → 네이티브 View로 변환되어 화면에 렌더링됨
- 새로운 아키텍처 (New Architecture - JSI)
  - 2024년 기준 공식적으로 도입된 새로운 아키텍처로, 기존의 비동기 브릿지를 JSI(JavaScript Interface)로 대체함
  - JavaScript가 C++ 객체의 참조를 직접 가져와 동기적으로 네이티브 메서드를 호출할 수 있음 (직렬화 비용 제거)
  - 결과적으로 브릿지 병목 현상이 사라져 네이티브 통신 속도와 UI 렌더링 성능이 크게 향상됨
- React Native 렌더링 과정 (Fabric)
  1. **JavaScript Layer**: 개발자가 JSX로 UI 구조(`<View>` 등) 정의
  2. **Shadow Tree 생성**: React의 재조정(Reconciliation) 과정을 통해 메모리에 Shadow Tree를 구축하고, Yoga 엔진을 통해 각 요소의 레이아웃(크기, 위치) 계산
  3. **Fabric 전달**: 완성된 Shadow Tree 정보를 네이티브 플랫폼에 전달
  4. **Native Mount**: 각 플랫폼(iOS/Android)에서 실제 네이티브 View를 생성하고 화면에 렌더링

### JSX → 네이티브 변환 예시

```jsx
<View>
  <Text>Hello World!</Text>
</View>
```

- iOS: `<UIView>`, `<UILabel>` 같은 UIKit 컴포넌트로 변환되어 렌더링됨
- 안드로이드: `<ViewGroup>`, `<TextView>` 같은 네이티브 View로 변환됨
