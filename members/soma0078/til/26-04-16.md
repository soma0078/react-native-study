## TanStack Query v5 + ky

`useQuery`는 서버 데이터 페칭에 필요한 로딩·에러·캐싱 상태를 자동으로 관리해줌. 주요 옵션:

- `staleTime` — 지정 시간 동안 캐시를 신선한 것으로 간주해 재요청하지 않음 (날씨는 10분 설정)
- `retry` — 요청 실패 시 자동 재시도 횟수
- `enabled` — `false`이면 쿼리를 실행하지 않음. 조건부 요청이나 쿼리 간 의존 관계 표현에 활용
- `gcTime` — 캐시를 메모리에서 유지하는 시간. 기본 5분. `staleTime`과 별개 개념
- `select` — 응답 데이터를 변환/필터링해서 반환. 컴포넌트가 필요한 값만 구독하게 할 수 있음
- `placeholderData` — 실제 데이터가 오기 전 임시로 보여줄 값. 이전 데이터를 유지할 때 `keepPreviousData` 패턴으로 활용
- `refetchInterval` — 지정한 주기마다 자동으로 재요청. 실시간성이 필요한 데이터에 활용

`ky`는 fetch 기반 HTTP 클라이언트로, `ky.create()`로 인스턴스를 만들면 baseURL·timeout·공통 파라미터를 한 곳에서 관리할 수 있음

| 비교           | fetch + useEffect | TanStack Query      |
| -------------- | ----------------- | ------------------- |
| 캐싱           | 직접 구현         | staleTime으로 자동  |
| 재시도         | 직접 구현         | retry 옵션          |
| 로딩·에러 상태 | useState 다수     | isLoading, isError  |
| 중복 요청 방지 | 직접 구현         | 동일 key 자동 dedup |

### Fallback Query 패턴

`enabled` 옵션으로 두 쿼리 사이에 의존 관계를 선언할 수 있음. 기본 쿼리가 실패했을 때만 폴백 쿼리를 활성화해서, 컴포넌트는 분기 없이 단일 값만 소비함. 위치 권한 거부 시 서울 시청 좌표로 자동 전환하는 데 활용함

```ts
const fallback = useQuery({
  ...
  enabled: primary.isError || lat == null, // 기본 쿼리 실패 시에만 활성화
});

return primary.isError || lat == null ? fallback : primary;
```

## KMA 기상청 API

기상청 단기예보 API는 위경도 대신 자체 격자 좌표(nx, ny)를 사용함. `kma-grid` 라이브러리로 위경도를 격자 좌표로 변환함. 예보는 하루 8번(02·05·08·11·14·17·20·23시) 발표되고 발표 후 약 10분 뒤부터 접근 가능하기 때문에, 현재 시각 기준으로 가장 최근 발표 시각을 계산하는 로직이 필요함

체감온도는 기온에 따라 공식을 분기함:

- 10°C 이상 → 열지수 (Steadman 공식, 기온·습도 사용)
- 10°C 미만 → 풍속냉각 (캐나다 기상청 공식, 기온·풍속 사용)

## React Native 애니메이션: Animated API vs Reanimated

| 항목      | Animated API (내장)                 | Reanimated v4 (라이브러리) |
| --------- | ----------------------------------- | -------------------------- |
| 실행 환경 | JS 스레드                           | UI 스레드                  |
| 성능      | 복잡한 애니메이션 시 드롭 발생 가능 | 60fps 안정적 유지          |
| 제스처    | Pressable만 지원                    | Gesture API로 세밀한 제어  |
| 색상 보간 | 지원 안 함                          | interpolateColor 지원      |

**핵심 차이**: Animated API는 JS 스레드에서 계산해 JS가 바쁘면 끊김 발생. Reanimated는 UI 스레드(Worklets)에서 처리해 JS 무관하게 60fps 유지

## Reanimated v4

### Worklets과 runOnJS

- **Worklets**: JS 코드를 네이티브 UI 스레드에서 실행 가능하도록 변환한 함수. 애니메이션 계산이 JS 스레드를 거치지 않아 JS 브리지 지연 제거. `useAnimatedStyle` 등 Reanimated 훅 내에서 자동으로 처리됨
- **runOnJS**: Worklet 내에서 일반 JS 함수(`setState`, `alert` 등)를 호출할 때 필요. UI 스레드 → JS 스레드로 컨텍스트 전환. 예: 제스처 종료 시 상태 업데이트하면서도 애니메이션은 부드럽게 유지

### 애니메이션 함수 종류

| 함수           | 특징                                        | 사용 시기                       |
| -------------- | ------------------------------------------- | ------------------------------- |
| `withTiming`   | 지정한 duration 동안 선형(또는 easing) 변화 | 정확한 시간 제어가 필요할 때    |
| `withSpring`   | 물리 기반 스프링. 자연스럽게 튕김           | 제스처 반응, 카드 scale 등      |
| `withDecay`    | 초기 속도에서 마찰로 서서히 감속            | 스크롤·플릭 제스처 후 관성 표현 |
| `withSequence` | 여러 애니메이션을 순서대로 실행             | 흔들림(shake) 같은 복합 동작    |
| `withRepeat`   | 애니메이션을 반복 실행                      | 로딩 pulse, 무한 루프 효과      |
| `withDelay`    | 지정한 시간 후 애니메이션 시작              | 순차 등장 효과                  |

### 색상 보간 배경

`interpolateColor(value, inputRange, outputColors)`는 숫자 값을 색상 배열에 보간해줌. 날씨 조건을 숫자 인덱스로 매핑한 뒤 `withTiming`으로 인덱스 값을 천천히 바꾸면, 배경색이 자연스럽게 전환됨

```ts
progress.value = withTiming(CONDITION_INDEX[condition], { duration: 800 });

useAnimatedStyle(() => ({
  backgroundColor: interpolateColor(progress.value, INPUT_RANGE, COLORS),
}));
```

### Gesture API + Haptics

`Gesture.Tap()`은 탭 제스처의 생명주기를 세분화해서 제어할 수 있음:

- `onBegin` — 손가락이 닿는 순간
- `onEnd` — 탭 성공 시
- `onFinalize` — 성공·실패 관계없이 제스처가 끝날 때

워크렛(UI 스레드)에서는 JS 함수를 직접 호출할 수 없어 `runOnJS`로 감싸야 함. `Haptics.impactAsync`도 JS 함수이므로 동일하게 처리함. `expo-haptics`의 `ImpactFeedbackStyle`은 `Light` / `Medium` / `Heavy` 세 가지 강도를 제공함

이 외에 사용 가능한 제스처 타입:

| 제스처                | 설명                                       |
| --------------------- | ------------------------------------------ |
| `Gesture.Pan()`       | 드래그. `onUpdate`에서 dx·dy로 이동량 추적 |
| `Gesture.Pinch()`     | 두 손가락 핀치. scale 값 제공              |
| `Gesture.Rotation()`  | 두 손가락 회전. rotation 값 제공           |
| `Gesture.LongPress()` | 일정 시간 이상 누르고 있을 때              |

여러 제스처를 조합할 때는 `Gesture.Simultaneous()`(동시 인식)나 `Gesture.Exclusive()`(하나만 인식)로 우선순위 지정

`expo-haptics`의 다른 API:

- `notificationAsync(NotificationFeedbackType.Success | Warning | Error)` — 알림성 피드백, 성공·경고·오류 상황에 활용
- `selectionAsync()` — 선택 변경 시 미세한 피드백. 피커나 슬라이더에 적합

### 순차 등장 효과 예시

```ts
// index * 80ms씩 지연해 순차 등장
opacity.value = withDelay(index * 80, withTiming(1, { duration: 250 }));
ty.value = withDelay(
  index * 80,
  withSpring(0, { damping: 15, stiffness: 200 }),
);
```

`withSpring`은 `damping`(감쇠)·`stiffness`(강성)로 튀는 정도를 조절

## expo-location

### 권한 관련 메서드

| 메서드                              | 설명                                                                 |
| ----------------------------------- | -------------------------------------------------------------------- |
| `requestForegroundPermissionsAsync` | 권한 다이얼로그를 띄워 사용자에게 요청. 최초 1회만 다이얼로그 노출됨 |
| `getForegroundPermissionsAsync`     | 현재 권한 상태만 조회. 다이얼로그 없음                               |
| `requestBackgroundPermissionsAsync` | 백그라운드 위치 권한 요청. Foreground 권한 이후에 요청해야 함        |

권한 상태는 `granted` / `denied` / `undetermined` 세 가지로 반환됨.

**주의**: `getForegroundPermissionsAsync`는 권한을 요청하지 않으므로, 앱 최초 실행 시 `undetermined` 상태가 반환됨. 위치를 사용하는 훅/컴포넌트에서 직접 `requestForegroundPermissionsAsync`를 호출해야 Race Condition이 발생하지 않음

```ts
// 권한 요청과 위치 조회를 같은 곳에서 처리
const { status } = await Location.requestForegroundPermissionsAsync();
if (status !== "granted") return;
const { coords } = await Location.getCurrentPositionAsync(...);
```

### 위치 조회 메서드

| 메서드                      | 설명                                              |
| --------------------------- | ------------------------------------------------- |
| `getCurrentPositionAsync`   | 현재 위치를 한 번 조회. GPS 픽스가 필요함         |
| `getLastKnownPositionAsync` | 캐시된 마지막 위치를 반환. GPS 픽스 없이도 동작함 |
| `watchPositionAsync`        | 이동을 지속적으로 추적. 구독 형태로 콜백을 받음   |

`accuracy` 옵션으로 정확도와 배터리 소모를 조절할 수 있음:

| 옵션                          | 설명                                                                  |
| ----------------------------- | --------------------------------------------------------------------- |
| `Accuracy.Lowest` ~ `Highest` | 낮을수록 빠르고 배터리 소모 적음                                      |
| `Accuracy.Balanced`           | 일반적인 앱에 적합한 기본값. Android에서 Fused Location Provider 사용 |
| `Accuracy.BestForNavigation`  | 네비게이션 수준. 배터리 소모 큼                                       |

> Android에서 `Accuracy.Balanced` 이상은 Fused Location Provider를 사용하므로 Google Play Services가 필요함. AOSP / Google APIs 전용 에뮬레이터에서는 동작하지 않음

### 주소 변환 메서드

| 메서드                | 설명                                 |
| --------------------- | ------------------------------------ |
| `reverseGeocodeAsync` | 위경도 → 주소 객체 변환 (역지오코딩) |
| `geocodeAsync`        | 주소 문자열 → 위경도 변환 (지오코딩) |

`reverseGeocodeAsync`는 배열을 반환하며, 첫 번째 결과에 `district`(구), `city`(시), `region`(도/주) 등의 필드가 포함됨

```ts
const [place] = await Location.reverseGeocodeAsync({ latitude, longitude });
const locationName = place?.district ?? place?.city ?? null;
```

### app.json 설정

expo-location을 사용하려면 `app.json`의 plugins에 선언하고, Android는 권한을 명시적으로 추가해야 함

```json
"plugins": [
  [
    "expo-location",
    {
      "locationAlwaysAndWhenInUsePermission": "위치 권한 사용 이유 메시지"
    }
  ]
]
```

**`bundleIdentifier` / `package`**

앱의 고유 ID. OS가 앱을 식별하는 데 사용함. 역DNS 형식(`com.회사.앱명`)을 관례로 사용하며, 앱스토어/플레이스토어 배포 후에는 변경 불가함

| 필드               | 플랫폼  |
| ------------------ | ------- |
| `bundleIdentifier` | iOS     |
| `package`          | Android |

**Android `permissions`**

Android는 앱이 사용할 권한을 빌드 시점에 미리 선언해야 함. iOS는 `Info.plist`에 자동 추가되지만 Android는 명시적으로 지정해야 함

| 권한                     | 설명                                          |
| ------------------------ | --------------------------------------------- |
| `ACCESS_COARSE_LOCATION` | 네트워크/WiFi 기반 대략적인 위치 (수백m 오차) |
| `ACCESS_FINE_LOCATION`   | GPS 기반 정밀 위치                            |
