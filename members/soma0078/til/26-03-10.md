## React Native 스타일링 방법

### 1. StyleSheet

- React Native에서 기본 제공하는 스타일링 방식
- 스타일을 컴포넌트 외부에 분리해 관리

```tsx
import { StyleSheet, View, Text } from "react-native";

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: "#f5f5f5",
    padding: 16,
  },
  title: {
    fontSize: 24,
    fontWeight: "bold",
  },
});
```

---

- 인라인 스타일과 달리 `StyleSheet.create()`를 사용하면 스타일이 한 번만 생성되어 성능상 이점이 있음

### 2. 인라인 스타일

- 컴포넌트에 직접 스타일 객체 작성
- 간단한 스타일에는 편리하지만, 코드가 길어지면 가독성 떨어지고 렌더마다 객체 생성하여 성능상 불리할 수 있음

```tsx
<View style={{ flex: 1, backgroundColor: "#f5f5f5", padding: 16 }}>
  <Text style={{ fontSize: 24, fontWeight: "bold" }}>Hello</Text>
</View>
```

---

### 3. Utility-first (Tailwind CSS)

- React Native는 CSS 엔진이 없기 때문에 Tailwind의 `className`를 직접 해석할 수 없음
- 따라서 RN에서 Tailwind를 사용하려면 `className`을 React Native 스타일 객체로 변환해야 함

**동작 원리**

NativeWind / Uniwind 모두 **빌드 타임 변환** 방식으로 동작함

1. Babel 또는 번들러 플러그인이 소스코드를 분석
2. `className="..."` 문자열 감지
3. Tailwind 클래스 → React Native `StyleSheet` 객체로 변환
4. 번들에 변환된 스타일이 포함됨
5. 런타임에서는 추가 파싱 없이 바로 스타일 적용

```
"flex-1 bg-gray-100"  →  { flex: 1, backgroundColor: '#f3f4f6' }
```

> `className`은 RN 기본 prop이 아니라, 라이브러리가 JSX transform으로 주입한 것임

**NativeWind vs Uniwind**

|               | NativeWind                              | Uniwind                                      |
| ------------- | --------------------------------------- | -------------------------------------------- |
| 기반          | Babel + Metro 플러그인                  | unistyles + Metro/Vite 플러그인              |
| Tailwind 버전 | v3 (v4는 pre-release)                   | Tailwind 4 전용                              |
| 설정          | `tailwind.config.js`                    | CSS `@import` 방식                           |
| 타입 지원     | `className`이 `string` (오타 감지 불가) | 유효한 클래스명으로 타입 제한 (오타 시 에러) |

- Tailwind v4는 JS 설정 파일 대신 CSS 파일 기반으로 바뀌었는데, Uniwind는 처음부터 이에 맞게 설계된 것이 차이점
