### 2026-03-03 (화) TIL

## 1. StyleSheet 활용

런타임에 스타일 객체를 한 번만 생성하여 메모리 낭비를 줄임. `StyleSheet.create`를 사용하면 객체가 재사용되어 가비지 컬렉션 부담이 적고, TypeScript 환경에서 강력한 타입 체크를 제공함

- **문법**: camelCase를 기본으로 하며, 인라인 스타일보다 구조적으로 깔끔함
- **성능**: 인라인 스타일처럼 렌더링마다 새로운 객체를 생성하지 않아 메모리 효율적이며, 네이티브 렌더러가 더 효율적으로 스타일을 참조할 수 있도록 구조화됨

## 2. 모바일 환경에 최적화된 Flexbox

웹 브라우저와 달리 모바일은 세로로 긴 화면이 기본임. 이에 따라 React Native의 Flexbox는 웹과 몇 가지 차이가 있음

- **세로 중심**: `flex-direction`의 기본값이 `column`임. 리스트 형태의 UI가 주를 이루는 모바일 특성이 반영됨
- **단순화된 설계**: `flexShrink`가 기본적으로 0인 등 예측 가능하게 동작함

## 3. Image 컴포넌트 활용

이미지 컴포넌트는 웹보다 조금 더 엄격하게 다뤄야 함

- **크기 지정의 필수성**: 네트워크 이미지는 반드시 `width`, `height`를 명시해야 함. Yoga 엔진은 렌더링 전 각 요소의 자리를 미리 확정해야 하므로, 크기 정보가 없으면 이미지가 표시되지 않거나 레이아웃이 틀어질 수 있음
- **해상도 대응**: 디바이스마다 다른 픽셀 밀도(Density)를 고려해 `@2x`, `@3x` 접미사를 붙인 리소스를 준비하면 시스템이 적절한 이미지를 자동으로 선택함
- **캐시 제어**: `cache` 속성을 통해 네이티브 플랫폼의 캐시 전략을 직접 제어할 수 있음

## 4. 내비게이션

모바일 앱은 웹의 '페이지 이동'보다 '화면 쌓기'에 가까운 개념임

### Expo Router

파일 기반 라우팅으로 Next.js와 유사한 폴더 구조를 가짐

- **Stack 내비게이션**: 화면 위에 새로운 화면을 쌓는 방식. 스택 밑에 이전 화면이 물리적으로 깔끔하게 남아있기 때문에 뒤로가기 버튼이 자동으로 생김
- **Tabs 내비게이션**: 하단 탭 바를 통해 주요 메뉴를 전환하는 방식

### React Navigation

Expo Router의 근간이 되는 라이브러리로, 더 세밀한 내비게이션 커스텀이 필요할 때 직접 제어할 수 있는 저수준 API를 제공함
