## ✅ 기본 과제 — 핵심 컴포넌트 & 레이아웃

### 사용한 컴포넌트 정리


| 컴포넌트         | 역할                | 사용 위치                 |
| ------------ | ----------------- | --------------------- |
| `View`       | 레이아웃 컨테이너         | 모든 화면                 |
| `Text`       | 텍스트 표시            | 모든 화면                 |
| `Image`      | 이미지 표시            | Home, Profile         |
| `ScrollView` | 스크롤 가능한 컨테이너      | About, Profile        |
| `FlatList`   | 가상화 리스트 (심화에서 변경) | Home                  |
| `Pressable`  | 터치 영역 (최신 권장)     | Home, Profile, Detail |


### StyleSheet 분리

모든 화면에서 스타일을 컴포넌트 밖 `StyleSheet.create()`로 분리.

```tsx
// 컴포넌트 아래에 스타일 분리
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: "#fefefe",
    justifyContent: "center",
    alignItems: "center",
  },
});
```

### Flexbox 레이아웃

```tsx
// 가로 정렬 — Home 헤더 (아바타 + 텍스트)
header: {
  flexDirection: "row",     // 가로 배치
  alignItems: "center",     // 세로 중앙
  padding: 20,
}

// 세로 정렬 + 중앙 — Profile 카드
container: {
  flexGrow: 1,
  alignItems: "center",     // 가로 중앙
  justifyContent: "center", // 세로 중앙
}

// 가로 정렬 + 양끝 배치 — 카드 헤더 (제목 + 뱃지)
cardHeader: {
  flexDirection: "row",
  justifyContent: "space-between",
  alignItems: "center",
}
```

## ✅ 심화 과제 1 — FlatList vs ScrollView 성능 차이

### 핵심 차이


| 항목     | ScrollView              | FlatList                                           |
| ------ | ----------------------- | -------------------------------------------------- |
| 렌더링 방식 | **모든** 자식을 한 번에 렌더링     | 화면에 **보이는 항목만** 렌더링 (가상화)                          |
| 메모리    | 항목 수에 비례하여 증가           | 일정 수준 유지                                           |
| 적합한 상황 | 항목 수가 적고 고정된 경우 (< 20개) | 항목 수가 많거나 동적인 경우                                   |
| 구분선·헤더 | 직접 구현                   | `ItemSeparatorComponent`, `ListHeaderComponent` 내장 |


### 적용 — Home 탭을 FlatList로 변환

```tsx
// Before: ScrollView + map
<ScrollView>
  {PROJECTS.map((project) => (
    <View key={project.id}>...</View>
  ))}
</ScrollView>

// After: FlatList
<FlatList
  data={PROJECTS}
  keyExtractor={(item) => item.id}
  ListHeaderComponent={<Header />}
  renderItem={({ item }) => <ProjectCard project={item} />}
/>
```

**FlatList 주요 props:**

- `data` — 렌더링할 배열
- `renderItem` — 각 항목의 렌더 함수
- `keyExtractor` — 고유 키 추출 (성능 최적화에 필수)
- `ListHeaderComponent` — 리스트 위에 고정 헤더
- `ListEmptyComponent` — 데이터가 비었을 때 표시

## ✅ 심화 과제 2 — Platform별 스타일 분기 처리

### Platform.select 사용

iOS는 `shadow`* 속성, Android는 `elevation`으로 그림자를 처리. `Platform.select`로 한 곳에서 분기.

```tsx
import { Platform, StyleSheet } from "react-native";

const styles = StyleSheet.create({
  card: {
    padding: 16,
    backgroundColor: "#fff",
    borderRadius: 12,
    // 플랫폼별 그림자
    ...Platform.select({
      ios: {
        shadowColor: "#64748b",
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.08,
        shadowRadius: 8,
      },
      android: {
        elevation: 3,
      },
    }),
  },
});
```

### Platform.OS로 값 분기

```tsx
// 상세 화면 상단 패딩 — iOS는 Safe Area 고려
scrollContent: {
  paddingTop: Platform.OS === "ios" ? 60 : 24,
}
```

### 정리


| 방식                | 용도                     |
| ----------------- | ---------------------- |
| `Platform.OS`     | 단순 조건 분기 (`=== "ios"`) |
| `Platform.select` | 플랫폼별 객체 통째로 분기         |
| `*.ios.tsx`       | 파일 단위로 플랫폼별 컴포넌트 분리    |

## ✅ 심화 과제 3 — Stack + Tab Navigation 중첩 구조

### 구조

```
Root Stack (app/_layout.tsx)
├── index.tsx              ← Redirect → /(tabs)
├── (tabs)/                ← Tab Navigator
│   ├── index.tsx          ← Home 탭
│   ├── about.tsx          ← About 탭
│   └── profile.tsx        ← Profile 탭
└── detail/[id].tsx        ← Stack 상세 화면 (모달)
```

### Root Layout — Stack에 탭과 상세 화면 공존

```tsx
// app/_layout.tsx
<Stack screenOptions={{ headerShown: false }}>
  <Stack.Screen name="index" />
  <Stack.Screen name="(tabs)" />
  <Stack.Screen name="detail/[id]" options={{ presentation: "modal" }} />
  <Stack.Screen name="+not-found" />
</Stack>
```

### 탭에서 Stack 화면으로 이동

```tsx
// app/(tabs)/index.tsx — Home 탭에서 상세로 이동
import { useRouter } from "expo-router";

const router = useRouter();

<Pressable onPress={() => router.push(`/detail/${item.id}`)}>
  <Text>자세히 보기 →</Text>
</Pressable>
```

### 상세 화면에서 파라미터 받기 + 뒤로가기

```tsx
// app/detail/[id].tsx
import { useLocalSearchParams, useRouter } from "expo-router";

const { id } = useLocalSearchParams<{ id: string }>();
const router = useRouter();

// 뒤로가기
<Pressable onPress={() => router.back()}>
  <Text>돌아가기</Text>
</Pressable>
```

### 핵심 포인트

- `presentation: "modal"` → 상세 화면이 아래에서 올라오는 모달 형태
- 탭 화면에서 `router.push()`로 Stack 화면 이동 → 탭바가 사라지며 전체 화면 표시
- `router.back()`으로 이전 탭 화면 복귀

## ✅ 심화 과제 4 — 레퍼런스 디자인 컴포넌트

### 구현한 UI — 프로젝트 상세 카드 (detail/[id].tsx)

Dribbble 스타일의 카드 상세 화면 구성:

- **뒤로가기 버튼** — `Pressable` + `FontAwesome` 아이콘
- **제목** — 큰 볼드 텍스트
- **기술 뱃지** — `flexDirection: "row"` + `flexWrap: "wrap"`으로 태그 나열
- **설명 카드** — 둥근 모서리 + 플랫폼별 그림자 (iOS shadow / Android elevation)
- **배운 점 리스트** — 커스텀 bullet + 텍스트 정렬

```tsx
// 기술 뱃지 UI
<View style={styles.techRow}>
  {project.techs.map((tech) => (
    <View key={tech} style={styles.techBadge}>
      <Text style={styles.techText}>{tech}</Text>
    </View>
  ))}
</View>

// 배운 점 리스트
{project.learnings.map((item, index) => (
  <View key={index} style={styles.learningItem}>
    <View style={styles.bullet} />
    <Text style={styles.learningText}>{item}</Text>
  </View>
))}
```

## 전체 파일 구조 (최종)

```
app/
├── _layout.tsx          ← Root Stack
├── index.tsx            ← Redirect → /(tabs)
├── +not-found.tsx       ← 404
├── (tabs)/
│   ├── _layout.tsx      ← Tab Navigator (Home, About, Profile)
│   ├── index.tsx        ← Home — FlatList + Image + Pressable
│   ├── about.tsx        ← About — ScrollView + 스킬 리스트
│   └── profile.tsx      ← Profile — ScrollView + Image + Pressable
└── detail/
    └── [id].tsx         ← 상세 화면 (Stack 모달)
```

- `react-native`: `View`, `Text`, `Image`, `ScrollView`, `FlatList`, `Pressable`, `StyleSheet`, `Platform`, `Linking`
- `@expo/vector-icons`: `FontAwesome`
- `expo-router`: `Stack`, `Tabs`, `useRouter`, `useLocalSearchParams`, `Redirect`
