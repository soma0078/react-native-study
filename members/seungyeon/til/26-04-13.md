## ✅ Expo Router — 중첩 Stack과 네비게이션 히스토리 관리

### 파일 구조 = 라우팅 구조

```
app/
├── _layout.tsx          ← Root Stack
├── index.tsx            ← "/"
├── write/
│   ├── _layout.tsx      ← 중첩 Stack + Context Provider
│   ├── step1.tsx        ← "/write/step1"
│   └── step2.tsx        ← "/write/step2"
└── detail/
    └── [id].tsx         ← "/detail/:id" (동적 경로)
```

`write/` 폴더 안에 `_layout.tsx`를 두면 해당 폴더가 독립적인 Stack Navigator가 된다.  
step1 → step2 이동은 이 중첩 Stack 내부에서 처리되고, 폼 Context도 여기서 제공한다.

### router.push / navigate / replace 차이

| 메서드                  | 히스토리 동작                    | 주요 용도                   |
| ----------------------- | -------------------------------- | --------------------------- |
| `router.push(href)`     | 스택에 추가                      | 일반 화면 이동              |
| `router.navigate(href)` | 이미 있으면 재사용, 없으면 추가  | 탭 이동 등                  |
| `router.replace(href)`  | 현재 화면을 대체 (히스토리 제거) | 저장 완료 후 작성 화면 제거 |
| `router.back()`         | 스택에서 현재 화면 꺼냄          | 뒤로가기 버튼               |

### 저장 후 작성 화면이 히스토리에 남는 문제

`router.navigate("/")`로 이동하면 write 스택이 히스토리에 남아, 뒤로가기 시 step2가 다시 보였다.  
`router.replace()`로 write 스택 전체를 목적지 화면으로 대체해서 해결.

```tsx
// 목록으로 가기 — write 스택 전체를 홈으로 대체
const handleGoList = () => {
  router.replace("/");
};

// 일기 상세 보기 — write 스택 전체를 detail로 대체 (홈 경유 없이 바로 이동)
const handleViewDiary = () => {
  router.replace(`/detail/${savedId}`);
};
```

중간에 홈을 거쳐 detail로 push하는 2단계 방식은 버벅임이 생기므로, detail로 직접 replace하는 것이 자연스럽다.

## ✅ AsyncStorage — 로컬 데이터 영구 저장

기기 로컬에 문자열만 저장 가능하므로 JSON으로 직렬화/역직렬화해서 사용한다.

```ts
import AsyncStorage from "@react-native-async-storage/async-storage";

// 저장
await AsyncStorage.setItem("diaries", JSON.stringify(entries));

// 불러오기
const raw = await AsyncStorage.getItem("diaries");
const entries = raw ? JSON.parse(raw) : [];
```

**주의**: `AsyncStorage`는 비동기이므로 반드시 `await`를 사용해야 한다.  
큰 데이터나 복잡한 쿼리가 필요하면 SQLite (expo-sqlite) 또는 MMKV를 고려.

## ✅ useFocusEffect — 화면 포커스 시 데이터 갱신

`useEffect`는 컴포넌트 마운트 시 한 번만 실행된다.  
다른 화면에서 돌아왔을 때 목록을 갱신하려면 `useFocusEffect`를 사용한다.

```tsx
import { useFocusEffect } from "expo-router";
import { useCallback } from "react";

useFocusEffect(
  useCallback(() => {
    refresh(); // 화면에 포커스될 때마다 실행
  }, [refresh]),
);
```

`useCallback`으로 감싸지 않으면 매 렌더마다 새로운 함수가 생성되어 무한 루프가 발생한다.

## ✅ react-native-reanimated + GestureHandler — 스와이프 삭제

`Animated` (코어)와 달리 `reanimated`는 애니메이션을 JS 스레드가 아닌 UI 스레드에서 실행해 더 부드럽다.

```tsx
import { Gesture, GestureDetector } from "react-native-gesture-handler";
import Animated, {
  useSharedValue,
  useAnimatedStyle,
  withSpring,
  withTiming,
  interpolate,
  Extrapolation,
} from "react-native-reanimated";

const translateX = useSharedValue(0);

const panGesture = Gesture.Pan()
  .activeOffsetX([-10, 10]) // 가로 10px 이상 움직여야 제스처 시작
  .failOffsetY([-8, 8]) // 세로 8px 초과 시 스크롤로 넘김
  .onUpdate((e) => {
    translateX.value = Math.max(-90, Math.min(0, e.translationX));
  })
  .onEnd((e) => {
    translateX.value = e.translationX < -90 ? withTiming(-90) : withSpring(0);
  });

// 삭제 버튼 투명도 — translateX에 따라 자동 계산
const deleteAreaStyle = useAnimatedStyle(() => ({
  opacity: interpolate(
    translateX.value,
    [-90, -30, 0],
    [1, 0.5, 0],
    Extrapolation.CLAMP,
  ),
}));
```

| API                | 특징                                      |
| ------------------ | ----------------------------------------- |
| `useSharedValue`   | UI 스레드와 공유되는 애니메이션 값        |
| `useAnimatedStyle` | sharedValue 기반으로 스타일 자동 업데이트 |
| `withSpring`       | 스프링 물리 기반 애니메이션               |
| `withTiming`       | 일정 시간 동안 선형/이징 애니메이션       |
| `interpolate`      | 입력 범위 → 출력 범위 매핑                |

## ✅ Animated (코어) — 버튼 press 스케일 애니메이션

`useNativeDriver: true`를 사용하면 `transform`, `opacity` 속성은 네이티브 스레드에서 처리된다.

```tsx
const scale = useRef(new Animated.Value(1)).current;

const handlePressIn = () => {
  Animated.spring(scale, {
    toValue: 0.95,
    useNativeDriver: true,
    tension: 200,
    friction: 10,
  }).start();
};

const handlePressOut = () => {
  Animated.spring(scale, {
    toValue: 1,
    useNativeDriver: true,
    tension: 200,
    friction: 10,
  }).start();
};

<Animated.View style={{ transform: [{ scale }] }}>
  <Pressable onPressIn={handlePressIn} onPressOut={handlePressOut}>
    ...
  </Pressable>
</Animated.View>;
```

Modal 등장 애니메이션처럼 여러 값을 동시에 움직일 때는 `Animated.parallel`을 사용한다.

```tsx
Animated.parallel([
  Animated.spring(scale, { toValue: 1, useNativeDriver: true }),
  Animated.timing(opacity, {
    toValue: 1,
    duration: 200,
    useNativeDriver: true,
  }),
]).start();
```

## ✅ expo-haptics — 햅틱 피드백

사용자 인터랙션에 진동 피드백을 더해 조작감을 높인다.

```tsx
import * as Haptics from "expo-haptics";

// 버튼 탭 — 가벼운 진동
Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);

// 저장, 중요 액션 — 중간 진동
Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);

// 저장 완료 — 성공 알림 진동
Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);

// 삭제 확인 — 경고 진동
Haptics.notificationAsync(Haptics.NotificationFeedbackType.Warning);
```

## ✅ KeyboardAvoidingView — 키보드 올라올 때 레이아웃 보정

텍스트 입력 시 소프트 키보드가 입력창을 가리지 않도록 레이아웃을 자동으로 밀어 올린다.

```tsx
import { KeyboardAvoidingView, Platform } from "react-native";

<KeyboardAvoidingView
  style={{ flex: 1 }}
  behavior={Platform.OS === "ios" ? "padding" : "height"}
  keyboardVerticalOffset={Platform.OS === "ios" ? 0 : 20}
>
  {/* TextInput이 포함된 ScrollView */}
</KeyboardAvoidingView>;
```

- iOS: `"padding"` — 하단에 키보드 높이만큼 패딩 추가
- Android: `"height"` — 전체 높이를 줄여 키보드 위로 콘텐츠 배치
