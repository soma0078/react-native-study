## 1. 위치 권한 — `expo-location`

모바일 앱은 민감한 기기 기능(위치, 카메라, 알림 등)을 사용하기 전에  
**반드시 사용자에게 허락을 받아야 한다.** 허락 없이 접근하면 OS가 차단한다.

권한 흐름은 두 단계로 나뉜다.

```ts
// 1단계 — 현재 권한 상태 조회 (팝업 없음)
const { status } = await Location.getForegroundPermissionsAsync();
// "granted" | "denied" | "undetermined"

// 2단계 — 실제 권한 요청 (OS 팝업 뜸)
const { status } = await Location.requestForegroundPermissionsAsync();
```

**`undetermined`가 핵심이다.** 앱 첫 실행 시엔 아직 묻지 않은 상태.  
이때만 권한 안내 화면을 보여주고, 이미 `denied`이면 설정 앱으로 유도해야 한다.

```tsx
useEffect(() => {
  if (!isChecking && permissionStatus === "undetermined") {
    router.replace("/permission"); // 권한 안내 화면으로
  }
}, [isChecking, permissionStatus]);
```

앱 설정 파일에도 권한 사용 목적을 반드시 명시해야 한다.  
iOS는 이 문구가 OS 팝업에 그대로 노출된다.

```json
// app.json
"ios": {
  "infoPlist": {
    "NSLocationWhenInUseUsageDescription": "현재 위치를 기반으로 날씨 정보를 가져옵니다."
  }
},
"android": {
  "permissions": ["ACCESS_FINE_LOCATION"]
},
"plugins": [
  ["expo-location", {
    "locationWhenInUsePermission": "현재 위치를 기반으로 날씨 정보를 가져옵니다."
  }]
]
```

## 2. 스플래시 스크린 — `expo-splash-screen`

앱이 처음 열릴 때 JS 번들 로딩 중에 잠깐 보이는 정적 화면.  
기본 동작은 번들 로드가 끝나는 즉시 자동으로 사라지는데,  
API 응답까지 기다렸다가 사라지게 하려면 직접 제어해야 한다.

```tsx
// 앱 진입 시 — 자동으로 사라지지 않게 막기
SplashScreen.preventAutoHideAsync();

// 데이터 준비가 됐을 때 직접 숨김
SplashScreen.hideAsync();
```

이 프로젝트에서는 네이티브 스플래시를 즉시 숨기고,  
그 자리에 커스텀 스플래시 컴포넌트(`SplashScreenView`)를 올린 뒤  
API 응답이 오면 페이드아웃하는 방식을 사용했다.

```
[앱 시작]
  → preventAutoHideAsync()  ← 네이티브 스플래시 유지
  → hideAsync()             ← 즉시 숨김 (커스텀으로 교체)
  → <SplashScreenView />    ← 리플 애니메이션 재생
  → API 응답 완료
  → 커스텀 스플래시 페이드아웃
  → 메인 화면 페이드인
```

## 3. 제스처 처리 — `PanResponder`

웹에서 `onMouseMove`로 드래그를 감지하듯,  
모바일에서는 터치 이벤트를 `PanResponder`로 처리한다.

```tsx
const panResponder = useRef(
  PanResponder.create({
    // 이 뷰가 제스처를 가로챌지 결정 — dx/dy는 터치 이동 거리
    onMoveShouldSetPanResponder: (_, { dx, dy }) =>
      Math.abs(dx) > 5 || Math.abs(dy) > 5,

    onPanResponderGrant: () => {
      // 터치 시작 — 눌림 효과
    },
    onPanResponderMove: (_, { dx }) => {
      // 드래그 중 — dx: 가로 이동량 (px)
      rotateY.setValue(dx * 0.02);
    },
    onPanResponderRelease: () => {
      // 손 뗌 — 원래 위치로 복귀
    },
  })
).current;

// Animated.View에 핸들러 연결
<Animated.View {...panResponder.panHandlers}>
```

**`rotateY` transform은 숫자가 아닌 문자열(`"10deg"`)이 필요하다:**

```tsx
rotateY: rotateY.interpolate({
  inputRange: [-3, 3],
  outputRange: ["-45deg", "45deg"],
});
```

## 4. 네이티브 드리븐 애니메이션 — `useNativeDriver`

웹에서 CSS 애니메이션이 GPU에서 돌듯,  
RN에서도 애니메이션을 JS 스레드가 아닌 **네이티브 스레드**에서 처리할 수 있다.

```tsx
Animated.timing(fadeAnim, {
  toValue: 1,
  duration: 500,
  useNativeDriver: true, // ← 네이티브 스레드에서 처리
}).start();
```

`useNativeDriver: true`를 쓰면 JS가 바빠도 애니메이션이 끊기지 않는다.  
단, **`opacity`와 `transform`만 지원**한다.

```tsx
// 가능 — opacity, transform(scale, translate, rotate)
style={{ opacity: anim, transform: [{ scale: anim }] }}

// 불가능 — backgroundColor, width, height 등 레이아웃 속성
// → useNativeDriver: false 로 써야 함 (NumberTicker에서 사용)
```

## 5. 터치 컴포넌트 — `Pressable` vs `TouchableOpacity`

웹의 `<button>`에 해당하는 컴포넌트들이다.

```tsx
// Pressable — 최신 권장 방식, pressed 상태를 style에서 직접 사용 가능
<Pressable
  onPress={handlePress}
  onPressIn={springIn}   // 터치 시작
  onPressOut={springOut} // 터치 끝
  style={({ pressed }) => [
    styles.button,
    pressed && { opacity: 0.7 }
  ]}
>
  <Text>버튼</Text>
</Pressable>

// TouchableOpacity — 구버전, 눌렸을 때 자동으로 투명도 조절
<TouchableOpacity onPress={handlePress} activeOpacity={0.7}>
  <Text>버튼</Text>
</TouchableOpacity>
```

## 6. 가로 스크롤 리스트 — `FlatList`

`FlatList`는 화면에 보이는 항목만 렌더링하는 **가상화 리스트**다.  
`ScrollView + map`은 모든 항목을 한 번에 렌더링하므로 항목이 많으면 느려진다.

```tsx
<FlatList
  data={activities}
  horizontal // 가로 스크롤
  showsHorizontalScrollIndicator={false}
  keyExtractor={(item) => item.id}
  renderItem={({ item }) => (
    <ActivityCard activity={item} onPress={onActivityPress} />
  )}
  contentContainerStyle={styles.list}
  snapToInterval={172} // 카드 하나씩 딱 멈추는 스냅
  decelerationRate="fast" // 스냅이 빠르게 작동하도록
/>
```

`snapToInterval`은 모바일에서 자주 쓰는 패턴으로,  
스크롤이 멈출 때 항상 카드 단위로 정렬되게 만든다.

## 7. 하단 시트 — `@gorhom/bottom-sheet`

하단 시트 UI는 모바일 앱의 대표적인 패턴이다.  
단순히 열고 닫는 용도라면 RN 기본 `Modal`로도 구현할 수 있지만,  
**드래그로 여러 높이에 멈추는 스냅 포인트**가 필요하면 라이브러리가 필요하다.

```tsx
const bottomSheetRef = useRef<BottomSheet>(null);
const snapPoints = useMemo(() => ["50%", "85%"], []);

// 코드로 열기/닫기
bottomSheetRef.current?.expand(); // 열기 (첫 번째 snapPoint)
bottomSheetRef.current?.close(); // 닫기

<BottomSheet
  ref={bottomSheetRef}
  index={-1} // -1 = 처음엔 닫혀있음
  snapPoints={snapPoints} // 드래그로 멈추는 위치들
  enablePanDownToClose // 아래로 드래그하면 닫힘
  onChange={(index) => {
    if (index === -1) onClose(); // 완전히 닫혔을 때
  }}
>
  <BottomSheetScrollView>{/* 내용 */}</BottomSheetScrollView>
</BottomSheet>;
```

라이브러리가 필요한 이유는 시트 안의 ScrollView와 시트 드래그 제스처가 충돌하기 때문이다.  
내부 스크롤이 최상단에 닿았을 때 그 다음 드래그를 시트 닫기로 전환하는 로직이  
`PanResponder`만으로는 구현하기 까다롭다. 라이브러리는 이걸 Reanimated + Gesture Handler로 해결한다.

**필수 설정:** 앱 최상단을 `GestureHandlerRootView`로 감싸야 한다.

```tsx
// app/_layout.tsx
import { GestureHandlerRootView } from "react-native-gesture-handler";

export default function RootLayout() {
  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      <Stack screenOptions={{ headerShown: false }} />
    </GestureHandlerRootView>
  );
}
```

## 8. 동적 배경 — `expo-linear-gradient`

모바일에서 CSS `background: linear-gradient`에 해당하는 컴포넌트.  
`View` 대신 `LinearGradient`를 쓰면 된다.

```tsx
import { LinearGradient } from "expo-linear-gradient";

<LinearGradient
  colors={["#0a1628", "#1565c0", "#1e88e5"]} // 시작→끝 색상
  start={{ x: 0, y: 0 }} // 시작점 (좌상단)
  end={{ x: 0.3, y: 1 }} // 끝점 (우하단 방향)
  style={styles.container}
>
  {/* 자식 컴포넌트 */}
</LinearGradient>;
```

이 앱에서는 날씨 상태(맑음/흐림/비)와 시간대(새벽/낮/노을/밤)에 따라  
`getGradientConfig()`에서 색상 배열을 계산해 배경을 동적으로 변경했다.

## 전체 모바일 기능 흐름

```
앱 시작
  ↓
[스플래시] expo-splash-screen으로 네이티브 스플래시 제어
  ↓
[권한 확인] expo-location으로 위치 권한 상태 조회
  ↓ undetermined이면
[권한 화면] OS 팝업 띄워 사용자에게 허락 요청
  ↓ 허락하면
[홈 화면]
  ├─ 기상청 API 호출 → LinearGradient로 날씨별 배경
  ├─ FlatList 가로 스크롤 카드 리스트
  ├─ PanResponder로 카드 드래그 제스처
  ├─ Animated API로 숫자 카운팅, 파동, 페이드인
  └─ BottomSheet로 활동 상세 화면 (스냅 포인트 50%, 85%)
```
