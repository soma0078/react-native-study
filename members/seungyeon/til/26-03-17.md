## ✅ Metro Bundler

### 개념

React Native 전용 JS 번들러. Next.js의 Webpack/Turbopack과 같은 역할이지만 모바일 특화로 동작한다.

- **웹**: 빌드 결과물을 브라우저가 네트워크로 받아 실행
- **RN**: Metro가 번들을 만들고 기기 내 **Hermes JS 엔진**이 직접 실행 (브라우저 없음)

### Metro가 하는 일

1. JSX / TypeScript → JS 트랜스파일
2. 모듈 그래프 분석 (`require` 추적)
3. 번들 캐싱 (변경된 파일만 재빌드)
4. Source map 생성 (디버깅용)

### 개발 서버 흐름

```
코드 저장 (⌘S)
  → Metro Dev Server (localhost:8081) — 번들 변환 & 서빙
  → 기기 / 시뮬레이터 (Hermes 실행)
  → Fast Refresh — 해당 컴포넌트만 교체, 상태 유지
```

### 유용한 단축키 (터미널)

| 키   | 동작                     |
| ---- | ------------------------ |
| `r`  | 앱 리로드                |
| `d`  | 개발자 메뉴 열기         |
| `j`  | 디버거 열기              |
| `⇧m` | Expo 메뉴 (Expo 사용 시) |

### 트러블슈팅

Fast Refresh가 안 먹히거나 새 패키지 설치 후 이상 동작 시 캐시 초기화.

```bash
# Expo
npx expo start --clear

# React Native CLI
npx react-native start --reset-cache
```

> Next.js의 `.next` 폴더 삭제와 같은 개념

## ✅ 터치 컴포넌트

### TouchableOpacity (레거시)

구버전 방식. 터치 시 투명도 낮아지는 피드백이 기본 내장. 레거시 코드에서 자주 보임.

```jsx
import { TouchableOpacity, Text } from "react-native";

<TouchableOpacity
  onPress={() => console.log("눌림")}
  onLongPress={() => console.log("길게 눌림")}
  activeOpacity={0.7} // 터치 시 투명도 (기본 0.2)
>
  <Text>버튼</Text>
</TouchableOpacity>;
```

### Pressable (현재 공식 권장)

`pressed` state를 콜백으로 받아 스타일과 자식 컴포넌트를 동적으로 제어할 수 있음.

```jsx
import { Pressable, Text, StyleSheet } from "react-native";

<Pressable
  onPress={() => console.log("눌림")}
  onPressIn={() => console.log("손가락 닿음")} // mousedown 느낌
  onPressOut={() => console.log("손가락 뗌")} // mouseup 느낌
  onLongPress={() => console.log("길게")}
  style={({ pressed }) => [styles.button, pressed && styles.pressed]}
>
  {({ pressed }) => <Text>{pressed ? "누르는 중" : "버튼"}</Text>}
</Pressable>;
```

```js
const styles = StyleSheet.create({
  button: {
    backgroundColor: "#6366f1",
    padding: 14,
    borderRadius: 8,
  },
  pressed: {
    backgroundColor: "#4f46e5",
    opacity: 0.9,
  },
});
```

### 이벤트 발생 순서

```
onPressIn → (유지 시) onLongPress
          → onPressOut → onPress
```

- `onPressIn` — 애니메이션 시작, 즉각 반응 필요한 UI
- `onPress` — API 호출, 화면 이동 등 실제 액션

### hitSlop — 터치 영역 확장

```jsx
<Pressable
  hitSlop={{ top: 10, bottom: 10, left: 10, right: 10 }}
  onPress={handlePress}
>
```

### 언제 무엇을 쓸까

| 상황                       | 선택               |
| -------------------------- | ------------------ |
| 새로 작성하는 코드         | `Pressable`        |
| 누르는 동안 UI 변화 필요   | `Pressable`        |
| 레거시 코드베이스 유지보수 | `TouchableOpacity` |

## ✅ 네비게이션

### React Navigation vs Expo Router

|          | React Navigation 7    | Expo Router           |
| -------- | --------------------- | --------------------- |
| 방식     | 코드로 Navigator 선언 | 파일 구조 = 라우팅    |
| 러닝커브 | 보통                  | Next.js 경험자엔 낮음 |
| 실무     | 기존 프로젝트 대부분  | 신규 프로젝트 트렌드  |

> 추천: React Navigation 구조 이해 → Expo Router 적용. 개념은 같고 문법만 다름.

### Stack Navigator

화면이 카드처럼 쌓이고, 뒤로 가면 걷힘.

```jsx
import { createStaticNavigation } from "@react-navigation/native";
import { createNativeStackNavigator } from "@react-navigation/native-stack";

// v7 Static API — screens 객체로 선언, NavigationContainer 불필요
const RootStack = createNativeStackNavigator({
  initialRouteName: "Home",
  screens: {
    Home: HomeScreen,
    Detail: DetailScreen,
  },
});

const Navigation = createStaticNavigation(RootStack);

export default function App() {
  return <Navigation />;
}
```

```jsx
// 화면 이동 — useNavigation 훅 사용 (v7 권장)
import { useNavigation, useRoute } from '@react-navigation/native';

export default function HomeScreen() {
  const navigation = useNavigation();

  // 이동 + 파라미터 전달 (Next.js router.push와 유사)
  navigation.navigate('Detail', { userId: 42 });

  // 뒤로 가기
  navigation.goBack();
}

export default function DetailScreen() {
  const route = useRoute();
  const { userId } = route.params;   // 파라미터 받기
}
```

### Tab Navigator

import { createBottomTabNavigator } from "@react-navigation/bottom-tabs";

const Tabs = createBottomTabNavigator({

### Expo Router — 코드 비교

같은 구조를 파일 기반으로 구현. Next.js App Router와 거의 동일.

```
📁 app/
  _layout.tsx        ← Root Layout
  📁 (tabs)/
    _layout.tsx      ← 탭 레이아웃
    index.tsx        ← 홈 화면 "/"
    profile.tsx      ← 프로필 "/profile"
  📁 detail/
    [id].tsx         ← 상세 화면 "/detail/1"
```

```jsx
// 📁 app/(tabs)/index.tsx
import { router } from 'expo-router';

export default function HomeScreen() {
  return (
    <Button title="상세로" onPress={() => router.push('/detail/1')} />
  );
}

// 📁 app/detail/[id].tsx
import { useLocalSearchParams } from 'expo-router';

export default function DetailScreen() {
  const { id } = useLocalSearchParams();
  return <Text>{id}</Text>;
}
```

**핵심 차이 요약**

|           | React Navigation 7              | Expo Router                |
| --------- | ------------------------------- | -------------------------- |
| 화면 등록 | `screens` 객체로 선언           | 파일 만들면 자동           |
| 이동      | `navigation.navigate('Detail')` | `router.push('/detail/1')` |
| 파라미터  | `route.params`                  | `useLocalSearchParams()`   |
| 동적 경로 | 없음 (파라미터로 처리)          | `[id].tsx` Next.js 문법    |

### TypeScript 타입 설정

Static API 사용 시 타입이 자동 추론되므로 별도 선언이 거의 불필요. 레거시 Dynamic API에서만 수동 선언 필요.

```tsx
// Dynamic API 사용 시에만 필요한 타입 선언
export type RootStackParamList = {
  Home: undefined;
  Detail: { userId: number };
};

// 전역 등록 — useNavigation()이 앱 전체에서 타입 추론
declare global {
  namespace ReactNavigation {
    interface RootParamList extends RootStackParamList {}
  }
}
```

### 인증 플로우 — 실무 핵심 패턴

`isLoggedIn` 상태에 따라 Navigator 자체를 분기. 뒤로 가기로 로그인 화면에 돌아올 수 없는 효과.

```jsx
// 런타임 조건 분기가 필요하므로 Dynamic API 사용
const Stack = createNativeStackNavigator();

export default function App() {
  const { isLoggedIn } = useAuthStore();

  return (
    <NavigationContainer>
      <Stack.Navigator screenOptions={{ headerShown: false }}>
        {isLoggedIn ? (
          <Stack.Screen name="Main" component={TabNavigator} />
        ) : (
          <>
            <Stack.Screen name="Login" component={LoginScreen} />
            <Stack.Screen name="SignUp" component={SignUpScreen} />
          </>
        )}
      </Stack.Navigator>
    </NavigationContainer>
  );
}
```

> 인증 플로우처럼 런타임 분기가 필요하면 Dynamic API, 나머지는 Static API. 섞어 써도 OK.

### 전체 앱 구조 (실무 표준)

```
Root Stack Navigator
├── [미로그인] Auth Stack
│   ├── Login
│   └── SignUp
└── [로그인] Tab Navigator
    ├── 홈 탭 Stack
    │   ├── Home
    │   └── Detail
    └── 프로필 탭 Stack
        ├── Profile
        └── Edit
```
