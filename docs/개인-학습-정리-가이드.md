# 2026-02-15 TIL

## 오늘 배운 것
### useEffect 의존성 배열
- 빈 배열: 마운트 시 1회만 실행
- 값 포함: 해당 값 변경 시 실행
- 생략: 매 렌더링마다 실행

### 클린업 함수
```tsx
useEffect(() => {
  const subscription = subscribe();

  return () => {
    subscription.unsubscribe(); // 클린업
  };
}, []);
```

## 실수했던 것
- 의존성 배열에 함수를 넣었더니 무한 루프
- useCallback으로 함수 메모이제이션 필요

## 해결 방법
```tsx
const handlePress = useCallback(() => {
  // ...
}, [dependency]);

useEffect(() => {
  handlePress();
}, [handlePress]);
```

## 참고
- [useEffect 완벽 가이드](https://overreacted.io/a-complete-guide-to-useeffect/)

#### button-pattern.tsx
```tsx
// 재사용 가능한 버튼 컴포넌트
import { TouchableOpacity, Text, StyleSheet } from 'react-native';

interface ButtonProps {
  title: string;
  onPress: () => void;
  variant?: 'primary' | 'secondary';
}

export const Button = ({ title, onPress, variant = 'primary' }: ButtonProps) => {
  return (
    <TouchableOpacity
      style={[styles.button, styles[variant]]}
      onPress={onPress}
    >
      <Text style={styles.text}>{title}</Text>
    </TouchableOpacity>
  );
};

const styles = StyleSheet.create({
  button: {
    padding: 16,
    borderRadius: 8,
    alignItems: 'center',
  },
  primary: {
    backgroundColor: '#007AFF',
  },
  secondary: {
    backgroundColor: '#6C757D',
  },
  text: {
    color: '#FFFFFF',
    fontSize: 16,
    fontWeight: '600',
  },
});
```

#### api-fetch-pattern.ts
```tsx
// API 호출 패턴
import { useState, useEffect } from 'react';

export const useFetchData = <T,>(url: string) => {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      setLoading(true);
      try {
        const response = await fetch(url);
        const json = await response.json();
        setData(json);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]);

  return { data, loading, error };
};
```

# JavaScript 변수

## const vs let vs var

### const (추천)
- 재할당 불가
- 대부분의 경우 사용

```js
const name = 'Kim';
name = 'Lee'; // ❌ 에러!
```

### let
- 재할당 가능
- 값이 변경되는 경우만 사용

```js
let count = 0;
count = 1; // ✅ OK
```

### var (사용하지 말 것)
- 구식 문법
- 문제가 많음
